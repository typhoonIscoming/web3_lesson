# 第一部分：存储位置选择决策
- 1.1 为什么选择正确的存储位置很重要？
---

>> 在Solidity开发中，选择错误的存储位置会导致：

成本问题：
不必要的数据复制（memory vs calldata）

过多的气体消耗（存储操作）

用户交易成本增加

性能问题：

合约执行效率低下

区块链网络负担

用户体验下降

安全问题：
可能导致意外的修改状态
数据一致性问题

| 特性 | 贮存 | 记忆 | 调用数据 |
| :--- | :--: | ---: | :--: |
| 存储时长 | 永久保存 | 函数执行期间 | 函数执行期间 |
| 可修改性 | 确实可写 | 确实可写 | 尴尬 |
| Gas费用 | 最高 | 中等 | 最低 |
| 典型用途 | 状态变量 | 临时数据 | 外部参数 |
| SLOAD成本 | 2100+ 气体 | - | - |
| SSTORE成本 | 20,000 气体（首次） | - | - |
